# Contributing to Supply Sync

Thank you for your interest in contributing!

- Fork the repository and create a feature branch (e.g., `feat/new-ui`).
- Run tests and linters before opening a PR:

```powershell
flutter pub get
flutter test
```

- Open a pull request with a clear description and link to any related issue.
- Make sure your code follows project lint rules (`flutter_lints` is enabled).

See the GitHub PR template or open an issue first if you are unsure about a large change.
